# Feature Requests

Use this backlog for requested behavior that has not yet been approved for implementation. A request is not a commitment or proof that the feature is absent.

## Candidate requests requiring verification

- Add automated regression coverage for shortage entry, editing, copy output, similarity matching, ordering, receiving, transfers, and updates.
- Consolidate duplicated runtime logic only through a separately approved, compatibility-tested refactor.
- Add a documented, migration-safe representation for an empty supplier while continuing to read legacy records.
- Add diagnostics for sync state and last successful Firebase synchronization without exposing sensitive configuration or shop data.
- Improve accessible labels, focus behavior, and screen-reader feedback after a dedicated mobile accessibility review.
